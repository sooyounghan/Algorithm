### 시간 복잡도
-----
1. 알고리즘에서 시간 복잡도는 주어진 문제를 해결하기 위한 연산 횟수를 의미
2. 일반적으로 수행 시간은 1억 번의 연산을 1초의 시간으로 간주하여 예측

-----
### 시간 복잡도 정의
-----
1. 빅-오메가(Ω(n)) : 최선의 경우(Best Case)의 연산 횟수를 나타낸 표기법
2. 빅-세타(θ(n)) : 보통의 경우(Average Case)의 연산 횟수를 나타낸 표기법
3. 💡 빅-오(O(n)) : 최악의 경우(Worst Case)의 연산 횟수를 나타낸 표기법
4. 예시 : 0 ~ 99 사이의 무작위 값을 찾아 출력하는 코드
   - 빅-오메가(Ω(n)) 표기법의 시간 복잡도 : 1번
   - 빅-세타(θ(n)) 표기법의 시간 복잡도 : n / 2번
   - 빅-오(O(n)) 표기법의 시간 복잡도 : n번
```java
public class timeComplexityExample1 {
    public static void main(String[] args) {
        // 1 ~ 100 사이의 값 무작위 선택
        int findNumber = (int)(Math.random() * 100);
        for(int i = 0; i < 100; i++) {
            if(i == findNumber) {
                System.out.println(i);
                break;
            }
        }
    }
}
```
   - 빅-오메가(Ω(n)) 표기법의 시간 복잡도 : 1번
   - 빅-세타(θ(n)) 표기법의 시간 복잡도 : n / 2 = 100 / 2 = 50번
   - 빅-오(O(n)) 표기법의 시간 복잡도 : n = 100번

-----
### 빅-오 표기법(O(n))의 시간 복잡도
-----
1. 일반적으로 빅-오(O(n)) 표기법을 기준으로 수행 시간을 계산하는 것이 좋음

### 시간 복잡도 활용하기
-----
1. 알고리즘 선택의 기준으로 사용
   - 버블 정렬의 시간 복잡도 : O(n²)
   - 병합 정렬의 시간 복잡도 : O(n*log n)

2. 예제 - 수 정렬하기

  - 데이터의 개수 : N (1 ≤ N ≤ 1,000,000)
  - 제한 시간 : 2초

  - 시간 제한이 2초이므로 이 조건을 만족하려면 2억 번의 연산 횟수로 문제를 해결
  - 따라서, 문제에 주어진 시간 제한과 데이터 크기를 바탕으로 정렬 알고리즘으로 무엇을 사용할 것인지 판단해야함
  - 시간 복잡도는 항상 최악일 때, 즉 데이터 크기가 가장 클 때를 기준으로 함
  - 💡 연산 횟수 계산 방법 = 알고리즘 시간 복잡도 X 데이터의 크기
  - 알고리즘 적합성 평가
```
A. 버블 정렬 = (1,000,000)² = 1,000,000,000,000 > 200,000,000 : 부적합 알고리즘
B. 병합 정렬 = (1,000,000)log(1,000,000) = 약 20,000,000 < 200,000,000 : 적합 알고리즘
```

3. 이처럼 시간 복잡도 분석으로 사용 가능한 알고리즘 선택 가능
4. 즉, 데이터 크기(N)을 단서로 사용해야 하는 알고리즘 추측 가능

### 시간 복잡도를 바탕으로 코드 로직 개선
-----
1. 시간 복잡도는 작성한 코드의 비효율적 로직(Logic)을 개선하는 바탕으로도 사용 가능
2. 즉, 이 부분을 활용하면 가장 먼저 코드의 시간 복잡도를 도출할 수 있어야 함
3. 시간 복잡도를 도출하려면 다음 2가지 기준 고려
```java
A. 상수는 시간 복잡도 계산에서 제외
B. 가장 많이 중첩된 반복문의 횟수가 시간 복잡도의 기준이 됨
```
4. 예시
   - 연산 횟수가 N인 경우
```java
public class 시간복잡도_판별원리1 {
    public static void main(String[] args) {
        int N = 100000;
        int cnt = 0;
        for(int i = 0; i < N; i++) {
            System.out.println("연산 횟수 : " + cnt++);
        }
    }
}
```

  - 연산 횟수가 3N인 경우
```java
public class 시간복잡도_판별원리2 {
    public static void main(String[] args) {
        int N = 100000;
        int cnt = 0;
        for(int i = 0; i < N; i++) {
            System.out.println("연산 횟수 : " + cnt++);
        }

        for(int i = 0; i < N; i++) {
            System.out.println("연산 횟수 : " + cnt++);
        }

        for(int i = 0; i < N; i++) {
            System.out.println("연산 횟수 : " + cnt++);
        }
    }
}
```

  - 두 예제 코드의 연산 횟수는 3배의 차이가 나서, 큰 차이인 것 같지만 시간 복잡도는 일반적으로 상수를 무시
  - 따라서, 두 코드 모두 시간 복잡도는 O(n)

   - 연산 횟수가 N²인 경우
```java
public class 시간복잡도_판별원리3 {
    public static void main(String[] args) {
        int N = 100000;
        int cnt = 0;
        for(int i = 0; i < N; i++) {
            for(int j = 0; j < N; j++) {
                System.out.println("연산 횟수 : " + cnt++);
            }
        }
    }
}
```
  - 시간 복잡도는 가장 많이 중첩된 반복문을 기준으로 도출하므로 이중 for문이 전체 코드의 시간 복잡도 기준이 됨
  - 만약, 일반 for문 10개 더 있다 하더라도 도출 원리 기준 2번에 따라 시간 복잡도의 변화 없이 N²
