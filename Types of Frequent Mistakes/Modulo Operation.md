### 나머지 연산의 중요성
-----
1. 나머지 연산에는 자료형의 표현 범위를 넘지 않게 유도하고 나머지 연산의 원리를 알고 있는지 묻는 의도가 담겨져 있음
2. 나머지 연산의 분배 법칙 : 나눗셈을 제외하고, 덧셈 / 뺄셈 / 곱셈의 분배 법칙 성립

3. 문제를 구현할 때, 덧셈 / 뺼셈 / 곱셈을 모두 계산한 후 나머지 연산을 한 번만 수행하는 것이 아니라, 로직의 중간 과정마다 나머지 연산을 지속적으로 수행함으로써 자료형의 표현 범위를 초과하지 않도록 설계할 수 있다는 의미
   - 예) 1부터 50까지 곱한 값을 10007로 나눈 나머지라는 문제가 존재
```java
package frequentMistake;

public class mod {
    public static void main(String[] args) {
        long answer = 1;

        for(int i = 1; i <= 50; i++) {
            answer = answer * i;
        }

        System.out.println(answer % 10007); // 곱한 값을 100007으로 나머지 연산을 수행하는 로직
    }
}
```
   - 실행 결과
```
-6828
```
   - 자연수의 곱을 계산하고 나머지를 구했는데 의도치 않게 음수가 출력
   - 이는 자료형의 사용 문제이거나 로직의 문제일 수 있음
     + 로직을 살펴보면, 모든 값을 곱한 후 마지막에 나머지 연산을 수행 : 이 때문에 곱셈 과정에서 이미 long 자료형의 표현 범위를 초과하여 잘못된 값이 나오는 것
   - 이 문제는 나머지 연산에서 곱셈의 분배 법칙이 성립한다는 점을 활용하여 해결 가능 : 즉, 모든 값을 곱한 뒤 마지막에 나머지 연산을 한 번만 적용하는 것과 곱셈을 수행할 떄마다 나머지 연산을 적용하는 것이 동일한 결과를 낳음

```java
package frequentMistake;

public class mod2 {
    public static void main(String[] args) {
        long answer = 1;

        for(int i = 1; i <= 50; i++) {
            answer = (answer * i) % 10007; // 곱셈을 수행할 때마다 나머지 연산을 수행하는 로직
        }

        System.out.println(answer);
    }
}
```
   - 실행 결과
```
3235
```
   - 곱셉을 실행할 때마다 나머지 연산을 함께 수행하도록 코드를 수정하면, 결과값이 정상적으로 출력
   - 💡 일반적으로 문제에 '결과값을 OO로 나눈 나머지를 출력하세요'라는 문구가 존재한다면, 정답을 구한 후 마지막에 나머지 연산을 한 번만 적용하는 연산보다, 수행하는 과정마다 나머지 연산을 적용해 자료형의 표현 범위를 초과하지 않도록 구현해야 오답이 출력되는 것 방지 가능
