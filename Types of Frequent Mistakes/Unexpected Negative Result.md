### 예상치 못한 음수 결과 해결
-----
1. 문제를 해결하는 과정에서 종종 예상치 못한 음수 결과가 발생할 수 있음
   - 이는 주어진 조건이나 수식에 오류가 있거나, 변수의 자료형이 적절하지 않아 범위를 초과할 때 자주 나타남
   - 특히 자료형을 잘못 사용하는 경우가 많으므로, 이를 점검하고 필요에 따라 자료형을 변경하여 문제를 재검토하는 것이 중요

2. 실제 문제를 해결하는 과정에서 숫자 데이터를 다룰 때 가장 익숙한 int형을 자주 사용
   - 그러나 때떄로 int형이 잘못된 결과를 초래하는 원인이 될 수 있음
   - 결과값이나 연산 과정에서 int형이 표현할 수 있는 범위를 초과하는 경우가 발생하기 때문임
   - 이러한 상황에서는 자료형을 long형으로 변경함으로써 문제를 효과적 해결 가능

3. int형의 표현 범위를 초과하는 큰 숫자를 다룰 때는 long형을 사용하는 것이 안전
```java
package frequentMistake;

public class expressionRange {
    public static void main(String[] args) {
        int a = 1000000000;
        a += 2000000000; // 자료형의 표현 범위를 초과하는 연산
        System.out.println(a); 

        long b = 1000000000;
        b += 2000000000;
        System.out.println(b);
    }
}
```
   - a와 b라는 두 변수를 사용해 간단한 덧셈 연산을 하고 결과를 출력하는 코드
   - 실행 결과
```
-1294967296
3000000000
```

   - 그러나 int형으로 선언한 변수 a는 잘못된 결과 출력
   - 이처럼 코드에서 자료형을 잘못 설정하면 의도치 않은 결과가 나올 수 있으므로, 각 자료형의 표현 범위를 정확히 이해하고 문제에 맞는 자료형을 선택하는 것이 중요
