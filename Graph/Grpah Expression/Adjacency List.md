-----
### 그래프를 표현하기 - 인접 리스트
-----
1. ArrayList로 그래프를 표현
2. 노드 개수만큼 ArrayList를 선언하며, 자료형은 경우에 맞게 사용

-----
### 인접 리스트로 가중치 없는 그래프 표현하기
-----

1. 현재의 경우 Integer형이면 그래프를 표현하기 충분하므로 ```ArrayList<Integer>[5]```로 선언
2. 인접 리스트에는 N번 노드와 연결되어 있는 노드를 배열의 위치 N에 연결된 노드 개수만큼 배열에 연결하는 방식 표현
3. 예를 들어, 노드 1과 연결된 2, 3 노드는 ArrayList[1]에 [2, 3]을 연결하는 방식으로 표현 (방향이 있는 그래프를 표현)

-----
### 인접 리스트로 가중치 있는 그래프 표현하기
-----
1. 가중치가 있는 경우 자료형을 클래스로 사용
2. (도착 노드, 가중치)를 갖는 Node 클래스를 선언하여 ArrayList에 사용
<div align="center">
<img src="https://github.com/user-attachments/assets/20e7cc4b-b8a6-4a49-8aca-87558d72de58">
</div>

3. ArrayList[1]에 [(2, 8), (3, 3)]이 연결되어 있음
   - 이는 노드 1과 2가 가중치 8 에지로, 노드 1과 3이 가중치 3 에지로 연결되어 있음
   - 방향성도 고려되어 있음
  
4. 그래프를 구현하는 다른 방법에 비해 인접 리스트를 이용한 그래프 구현은 복잡한 편
5. 하지만 노드과 연결되어 있는 에지를 탐색하는 시간은 매우 뛰어나며, 노드 개수가 커도 공간 효율이 좋아 메모리 초과 에러도 발생하지 않음
6. 이런 장점으로 실제 코딩 테스트에서는 인접 리스트를 이용한 그래프 구현을 선호
