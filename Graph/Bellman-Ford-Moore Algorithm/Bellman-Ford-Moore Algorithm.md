2. 작동 원리
   - 에지 리스트로 그래프를 구현하고 최단 경로 배열 초기화
     + 벨만-포드 알고리즘은 엣지를 중심으로 동작하므로 그래프를 엣지 리스트로 구현
     + 또한, 최단 경로 배열은 출발 노드를 0, 나머지 노드는 무한대로 초기화
     + 출발 노드를 1로 선택해 벨만-포드 알고리즘을 진행 (Edge 클래스는 일반적으로 노드 변수 2개와 가중치 변수로 구성)
<div align="center">
<img src="https://github.com/user-attachments/assets/bf8bbb50-c461-47f5-beb0-889331b5c00e">
</div>

   - 모든 엣지를 확인해 정답 배열 업데이트
     + 최단 거리 배열에서 업데이트 반복 횟수는 노드 개수 - 1 : 노드 개수가 N이고, 음수 사이클이 없을 떄 특정 두 노드의 최단 거리를 구성할 수 있는 엣지의 개수는 N - 1이기 때문임
     + 특정 엣지 E = (s, e, w)에서 다음 조건을 만족하면 업데이트 실행 : D[s] != ∞ 이며, D[s] + w일 때, D[e] = D[s] + w로 배열의 값 업데이트
     + 음수 사이클이 없을 때, 최대 에지 개수가 나오려면 트리 형태에서 양 도착 노드를 선택해야 함
     + 엣지의 출발 노드를 s, 종료 노드를 e, 엣지의 가중치를 w로 가정
<div align="center">
<img src="https://github.com/user-attachments/assets/54bd6362-95b6-416e-a7de-dd6e7879bc3b">

   - 음수 사이클이 없을 때 N - 1번 엣지 사용 횟수를 반복하면, 출발 노드와 모든 노드 간 최단 거리를 알려 주는 정답 배열이 완성
   - 이렇게 완성 후 마지막으로 이 그래프에 음수 사이클이 존재하는지 확인해야 함

   - 음수 사이클 유무 확인하기
     + 음수 사이클 유무를 확인하기 위해 모든 엣지를 한 번씩 다시 사용해 업데이트 되는 노드가 발생하는지 확인
     + 만약 업데이트되는 노드가 있다면, 음수 사이클이 있다는 뜻이 되고, 2번째 단계에서 도출한 정답 배열이 무의미하고, 최단 거리를 찾을 수 없는 그래프라는 뜻이 됨
     + 음수 사이클이 존재하면, 이 사이클을 무한하게 돌수록 가중치가 계속 감소하므로 최단 거리를 구할 수 없음

   - 실제 코딩 테스트에서는 벨만-포드 알고리즘을 사용해 최단 거리를 구하는 문제보다 음수 사이클을 판별하는 문제가 더 빈번하게 출제
   - 따라서, 마지막에 한 번 더 모든 엣지를 사용해 업데이트되는 노드가 존재하는지 확인해야 함
