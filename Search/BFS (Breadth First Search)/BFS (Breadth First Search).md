### 너비 우선 탐색 (BFS, Breadth First Search)
-----
1. 그래프 완전 탐색 방법 중 하나로, 시작 노드에서 출발해 시작 노드를 기준으로 가까운 노드를 먼저 방문하면서 탐색하는 알고리즘

2. 너비 우선 탐색은 선입선출 방식으로 탐색하므로 큐를 이용해 구현
3. 시작 노드와 가까운 노드를 우선하여 탐색하므로 목표 노드에 도착하는 경로가 여러 개 일때 최단 경로를 보장
4. BFS의 원리
   - BFS를 시작할 노드를 정한 후 사용할 자료구조 초기화
     + BFS도 DFS와 마찬가지로 방문했던 노드는 다시 방문하지 않으므로 방문한 노드를 체크하기 위한 배열 필요
     + 그래프를 인접 리스트로 표현하는 것은 DFS와 동일
     + 차이점 : 탐색에 스택이 아닌 큐를 사용
     + 시작 노드를 큐에 삽입하며 방문 배열을 체크한 것을 보여줌
<div align="center">
<img src="https://github.com/user-attachments/assets/8b41274f-29a3-4e70-8584-461e3fcef98b">

   - 큐에서 노드를 꺼낸 후 꺼낸 노드의 인접 노드를 다시 큐에 삽입
     + 큐에서 노드를 꺼내면서 인접 노드를 큐에 삽입
     + 이 떄 방문 배열을 체크하여, 이미 방문한 노드는 큐에 삽입하지 않음
     + 또한, 큐에서 꺼낸 노드는 탐색 순서에 기록
     + 위 그림의 경우 1을 꺼내며 탐색 순서에 1을 기록하고 인접 노드 3, 2를 큐에 삽입하며 방문 배열에 체크 (3과 2는 아직 방문한 적 없음)

   - 큐 자로구조에 값이 없을 때 까지 반복하기
     + 큐에 노드가 없을 때 까지 앞선 과정을 반복
     + 선입선출 방식으로 탐색하므로 탐색 순서가 DFS와 다름

   - 2, 3 순서로 노드를 꺼내며 인접 노드를 큐에 삽입
     + 2의 경우 5, 을 아직 방문한 적이 없으므로 방문 배열을 체크하며 모두 삽입
     + 3의 경우 4 역시 방문한 적이 없으므로 방문 뱅려을 체크하며 삽입하므로, 탐색 순서는 2, 3이 기록

   - 5, 6을 꺼낼 때는 인접 노드가 없으니 탐색 순서에 기록만 하고 꺼냄
   - 4를 꺼낼 때는 인접 노드가 6이지만 이미 앞서 방문했으므로 6은 큐에 삽입하지 않고 꺼내기만 함
   - 최종 탐색 순서 : 1, 2, 3, 5, 6, 4
