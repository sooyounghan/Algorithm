-----
### 조합
-----
1. 코딩 테스트에서 자주 출제되는 주제이며, 동적 계획법을 이해하는 데 기초가 되는 매우 중요한 개념
2. 조합은 <img width="54" height="35" alt="image" src="https://github.com/user-attachments/assets/84c646f7-92bf-427e-8537-9046cd0b476e" />로 표현 : 이는 n개의 숫자에서 r개를 뽑는 경우의 수를 뜻함
3. 순열은 <img width="47" height="49" alt="image" src="https://github.com/user-attachments/assets/57676d54-3cb5-4fc7-b4f4-1da1443c929d" />로 표현 : n개의 숫자 중 r개를 뽑아 순서를 고려해 나열 경우의 수
4. 순열과 조합의 차이는 순서 고려 여부 : 즉, 조합에서는 데이터 1, 2, 3과 3, 2, 1을 같은 경우로 판단하고, 순열은 다른 경우로 판단
5. 순열의 수학 공식
<div align="center">
<img src="https://github.com/user-attachments/assets/e828e6e0-ef08-4d48-85fa-701ccd4d7b99">
</div>

   - 예를 들어, 5개 중 2개를 순서대로 선택하는 경우의 수를 구한다고 가정
     + 1번쨰 선택은 5개의 데이터를 선택할 수 있으므로 5가지를 선택할 수 있음
     + 2번째 선택은 1번째에서 선택한 데이터를 제외한 4가지를 선택할 수 있음
   - 따라서, 5개 중 2개를 순서대로 고르는 경우의 수는 5 * 4 = 20

6. 조합의 수학 공식
<div align="center">
<img src="https://github.com/user-attachments/assets/8c62ba8e-b7fd-47f1-b67a-d171ecd4c5e2">
</div>

  - 순열과 매우 비슷하며 분모에 r!만 추가된 것을 확인할 수 있음
  - r! : 순서가 다른 경우의 수를 제거하는 역할
  - 예를 들어, 5개 중 2개를 선택하는 경우의 수를 구한다고 가정
    + 기존 순열의 경우의 수에 2!로 나눠 5 * 4 / 2 = 10가지 경우의 수를 도출
    + 즉, 1과 2를 선택할 때와 2와 1을 선택할 때를 1가지 경우의 수로 만들기 위해 2로 나누는 것

7. 조합은 동적 계획법의 시작 : 따라서, 알고리즘에서 조합을 구현할 때는 수학 공식을 코드화하지 않고, 점화식을 사용해 표현
   - 특정 문제를 가정하기 : 적당한 조합 문제 가정 (5개의 데이터에서 3개를 선택하는 조합의 경우의 수를 푸는 문제)
<div align="center">
<img src="https://github.com/user-attachments/assets/11e73370-91da-4387-9efa-4c2af3da309e">
</div>

   - 모든 부분 문제가 해결된 상황이라고 가정하고 지금 문제 생각하기
     + 모든 부분 문제가 해결된 상황이라고 가정
     + 먼저 5개의 데이터 중 4개를 이미 선택이 완료된 데이터라고 가정하고, 마지막 데이터 선택 여부에 따라 경우의 수를 계산
     + 만약, 마지막 데이터를 포함해 총 3개의 데이터를 선택하려면 선택이 완료되었다고 가정한 4개의 데이터에서 2개를 선택해야 함
     + 마지막 데이터를 포함하지 않고 총 3개의 데이터를 선택하려면 이전 데이터 중 4개 중 3개를 선택해야 함
     + 2가지 경우의 수를 합치면 데이터 5개 중 3개를 선택하는 경우의 수가 나옴

   - 💡 모든 부분 문제가 해결된 상황이라고 가정하는 방법은 조합 뿐 아니라 동적 계획법에서도 꼭 필요

   - 점화식 표현
<div align="center">
<img src="https://github.com/user-attachments/assets/ad0c8b36-2c30-44a4-9747-51edb282e460">
</div>

   - 이 내용을 도출할 때, '4개 중 2개를 선택하는 경우의 수와 4개 중 3개를 선택하는 경우의 수를 구해야하는가?'를 고민해야 함
     + 하지만, 모든 부분 문제가 해결됐다고 가정했으므로, 지금은 5개 중 3개의 경우의 수를 구하는 것이 아니라 궁극적으로 조합과 관련된 점화식을 도출하는 것
     + 점화식을 정확하게 도출하면 부분 문제는 프로그램 로직을 이용해 자연스럽게 구할 수 있음

   - 특정 문제를 해결한 내용을 바탕으로 일반 점화식 도출하기 : 일반화된 점화식을 이용하면 조합과 관련된 모든 경우의 수를 쉽게 구할 수 있음
