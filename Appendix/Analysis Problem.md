### 전략 1 - 문제를 효과적으로 분석하기
-----
1. 출제자의 의도를 파악하여 어떤 프로그램을 만들어야 하는지 판단해야 함
   - 문제 내용이 단순하여 바로 의도를 파악하면 다행이지만, 지문이 길고 복잡해서 문제 자체를 명확하게 이해하지 못할 경우 당황할 수 있음
   - 이 때, 자칫하면 문제 의도를 모두 파악하지 못한 채 코딩을 시작하는 실수를 저지를 수 있음

2. 예제 - 수열 정렬하기
```
P[0], P[1], ..., P[N - 1]은 0부터 N - 1(포함)까지의 수를 한 번씩 완벽히 포함하는 수열이다.
수열 P의 길이가 N인 배열 A에 적용하면 길이가 N인 배열 B가 된다.
적용하는 방법은 B[P[i]] = A[i]이다.

배열 A가 주어졌을 때, 수열 P를 적용한 결과가 비 내림차순이 되는 수열 P를 찾는 프로그램을 작성하시오.
비 내림차순이란, 배열의 각 원소가 바로 앞에 있는 원소보다 크거나 같은 상태를 말한다.
만약 그러한 수열이 여러 개라면, 사전 순으로 앞서는 것을 출력한다.
```

  - 입력
    + 1번째 줄에 배열 A의 크기 N이 주어짐
    + 2번째 줄에 배열 A의 원소가 0번부터 차례대로 주어짐
    + N은 50보다 작거나 같은 자연수이고, 배열의 원소는 1,000보다 작거나 같은 자연수

  - 출력 : 1번째 줄에 비 내림차순으로 만드는 수열 P를 출력

3. 입력에 배열 A와 크기 N이 주어지고, 출력은 수열 P
   - 문제에서 B[P[i]] = A[i]라는 수식이 주어지더라도, 바로 이해하기는 어려움
   - 문제에 주어진 단서를 이용해 예제를 풀어봐야 함
     + 보통 예제는 복잡하지 않고 가장 기본적인 형태이므로 복잡한 계산 없이도 풀이 과정을 따라가기 쉬움
     + 내용을 눈으로 읽기보다 예제를 풀면서 역으로 문제를 파아하는 것이 빠르고 정확한 방법이 될 수 있음

   - 더불어 입력과 출력, 그리고 수식을 어떻게 사용하는지 자연스럽게 사용 가능

6. 예제 1번 : A배열을 오름차순으로 정렬하면 결국 B배열이 됨
   - 즉, A 배열을 오름차순 정렬하면 바로 B 배열을 구할 수 있음
   - 하지만, 궁극적으로 구해야 하는 것은 P 배열이므로, 따라서 B 배열을 오름차순 정렬로 구하되, 정렬할 때 기존 A 배열의 원래 인덱스 값을 함께 저장
     + 예를 들어, B 배열에서 B[0] = 1이라는 것을 정렬로 구할 수 있음
     + 이를 수식에 따라 역으로 생각하면 P[i] = 0, A[i] = 1이라는 것을 알 수 있음
     + 이제 i값을 찾으면 P 배열 일부가 완성 : 아까 기억해둔 A 배열 인덱스 값 사용
     + B 배열의 0번째와 매핑되는 A 배열의 인덱스는 1이므로, P[1] = 0이라는 P 배열 값을 구할 수 있음

7. 즉, 예제를 직접 풀어보는 과정에서 문제의 요구사항과 의도, 주어진 데이터들의 관계를 명확히 파악하는 것이 중요
   - 참고 : 오름차순 정렬 후 A 배열의 인덱스를 이용해 P 배열을 만드는 부분을 좀 더 생각해보면 다음과 같이 구현 가능
