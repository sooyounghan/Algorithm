### 탈락을 결정 짓는 실수 제거하기
-----
1. 코딩 테스트는 시험 때마다 환경과 시스템이 다름
   - 어떤 시험에서는 코드를 제출하면 바로 정답 여부를 알려 주고, 또 어떤 시험에서는 코드를 제출하면 채점하여 사후 결과를 알려주기도 함
   - 하지만, 검증 과정은 모든 형태의 코딩 테스트에서 꼭 수행해야 함

2. 평소 문제를 풀 때도 코드를 검증하는 습관과 역량을 길러 두면 실제 코딩 테스트에서 큰 도움이 됨
   - 예를 들어, 코드를 제출하면 정답 여부를 즉시 알려주는 환경이 있다고 가정
     + 작성한 코드가 예제 테스트에서 모두 정답을 출력하지만, 실제 제출했을 때 오답으로 판별되어 실패하는 경우 존재
     + 이럴 때, 검증 과정을 통해 잘못된 부분을 빠르게 찾아내고 수정할 수 있음

   - 사후 결과를 알려주는 코딩 테스트 : 검증의 중요성이 더욱 강조 (사소한 실수로 실패 결과를 받아 낙담하는 경우 존재)
